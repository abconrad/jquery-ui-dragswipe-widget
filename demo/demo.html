What is It?

DragSwipe is a small jQuery UI extension of the default Draggable plugin provided by the jQuery UI team. It allows you to fire events after a user has dragged an element a predefined distance. Because it is an extension of jQuery UI's Draggable plugin it also has all of the functionality and accepts all options of jQuery UI Draggable.

The jQuery UI Draggable and Droppable plugins are great for allowing you to raise events when the user drags an element to a prespecified area. What if you want to be able to raise events when a user drags an element a prespecified distance? You can almost do it by combining Draggable and Droppable with some creative HTML but it is hacky and doesn't work as well as I would like.

I designed DragSwipe to emulate the swipe functionality of notifications on Android devices. It does this well and is also flexible enough to have the potential to do much more.

Demos

Default Functionality

Enable dragswipe functionality on any DOM element. Move the dragswipe object by clicking on it left or right. When the element is dragged passed farther than 150px (the default swipe distance threshold) it enters swipe state.

You can define the elements swipe state style in CSS. Here is the CSS applied to all of the following demos.

CSS:


    .hip-dragswipe-state-swipe {
        color: #f00;
        background-color: #ffc0cb;
    }

DragSwipe toggles the hip-dragswipe-state-swipe class depending on whether it has been dragged past it's threshold.

Swipe Direction and Distance

Define the axis that the elements are allowed to be dragged. Set the axis option to limit the draggable's path to the x- or y-axis.

Define the swipe threshold distance by passing a value to the swipeDistance option. swipeDistance accepts a numeric number of pixels.

I will enter swiped state after being dragged 200 pixels in any direction.
I will enter swiped state after being dragged 100 pixels vertically.
I will enter swiped state after being dragged 50 pixels horizontally.

JavaScript:


    //Swipe Direction and Distance
    $(document).ready(function() {
        $('.any-200').dragswipe({
            axis: false
        });
        $('.vert-100').dragswipe({
            swipeDistance: 100,
            axis: 'y'
        });
        $('.hor-50').dragswipe({
            swipeDistance: 50
        });
    });

        </div>

            I will enter swiped state after being dragged 50

        </div>

Remove after Swipe

Remove an element from the unordered list by swiping left or right. Setting the removeOnSwipe to true causes the element to be removed from the DOM when it gets swiped.

  • Swipe to remove 1
  • Swipe to remove 2
  • Swipe to remove 3
  • Swipe to remove 4
  • Swipe to remove 5

JavaScript:


    //Remove after Swipe
    $(document).ready(function() {
        $('.demo3>li').dragswipe({
            swipeDistance: 75,
            removeOnSwipe: true
        });
    });

Custom Functionality after Swipe

Call a custom function after the element is swiped. This is done by passing swipe as a callback function.

I will run a custom method after swiped.

JavaScript:


    //Custom Functionality after Swipe
    $(document).ready(function() {
        $('.custom-event').dragswipe({
            axis: false,
            swipe: function (event, ui) {
                alert('I have been swiped!');
            }
        });
    });

HTML:


    <div class="box">
        <div class="item-box custom-event">
            I will run a custom method after swiped.
        </div>
    </div>
    <div style="clear:both"></div>

Custom Functionality after Swipe from Trigger

Swipe also triggers a swipe event.

jQuery UI prefaces all UI events with the name of the widget that fired the event in lowercase. To listen for the DragSwipe swipe event you must actually listen for dragswipeswipe.

Some other JavaScript is listening for my swipe.

JavaScript:


    //Default DragSwipe initialization
    $('.custom-trigger').dragswipe();

    //Listens for the swipe event
    $('.custom-trigger').on(
        'dragswipeswipe',
        function (e) {
            alert('The 5th demo was swiped!');
        }
    );
Click here for more on listening for events using jQuery.

Options

Option Description Default
addClasses If set to false, will prevent the ui-draggable class from being added. This may be desired as a performance optimization when calling .draggable() on hundreds of elements. true
axis Constrains dragging to either the horizontal (x) or vertical (y) axis. Possible values: "x", "y". 'x'
removeOnSwipe Removes the element from the DOM if set to true false
revertDuration The duration of the revert animation, in milliseconds. Ignored if the revert option is false. 200
swipeDistance Threshold distance, in pixels, the element can be dragged before it enters the swipe state. 150

Click here for a list of the options provided by jQuery UI Draggable.

Events

Event Description
create Fires when the widget is created for the first time.
swipe Fires when a DragSwipe element is dropped while in a swipe state (based on swipeDistance option).

Click here for a list of the events provided by jQuery UI Draggable.
